Create `.github/workflows/ci.yml` with this EXACT content:

```yaml
# .github/workflows/ci.yml
# Meta-repo CI — gates every PR against main.
# Runs lint-all, typecheck-all, build-all, test-all, and changeset:status.
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

concurrency:
  group: ci-${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  verify:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: pnpm/action-setup@v4.4.0
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build all packages
        run: pnpm run build-all

      - name: Typecheck
        run: pnpm run typecheck-all

      - name: Lint
        run: pnpm run lint-all

      - name: Test
        run: pnpm run test-all

      - name: Changesets status (PR only)
        if: github.event_name == 'pull_request'
        run: pnpm run changeset:status
```

Key decisions embedded:
- `fetch-depth: 0` is REQUIRED so `changeset status --since=origin/main` can diff against history (RESEARCH pitfall #2).
- `build-all` runs BEFORE typecheck because downstream packages (e.g., `@minion-stack/auth` depending on `@minion-stack/db`) need upstream `dist/` to typecheck cleanly (RESEARCH Pattern 1).
- `cache: 'pnpm'` warms installs via pnpm's content-addressable store.
- `concurrency` cancels stale runs on force-push.
- `changeset:status` only runs on PR events (not on `push: main`) — a merged changeset no longer exists at the main branch HEAD.
- pnpm version pinned to major `10` — matches `packageManager: pnpm@10.15.0` via Corepack, per RESEARCH A6.

Do not add branch protection config, dependabot, matrix jobs, or release job to this file — those are out of scope for POLISH-01. Release workflow lives in plan 08-02.

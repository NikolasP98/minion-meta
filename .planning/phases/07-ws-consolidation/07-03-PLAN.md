## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Consumer -> gateway server (WS) | All three consumers cross the untrusted network to the gateway. Auth tokens and device-signed payloads travel in the connect request; responses are treated as trusted once the connect handshake completes. |
| npm registry -> consumer build | Consumers pull @minion-stack/shared@^0.3.0 from npm; bundle integrity enforced by lockfile + npm signatures. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08 | Spoofing | Device identity in paperclip onChallenge | mitigate | signDevicePayload (Ed25519 via node:crypto) stays in src/shared/device-auth.ts and executes inside buildConnectParams. Nonce comes from server connect.challenge. |
| T-07-09 | Tampering | Hub bundle accidentally pulls Node-only `ws` | mitigate | Only import from '@minion-stack/shared' root (not /node) in hub + site; verify via `bun run build` inspecting for "ws" externalization warnings. Listed as acceptance criterion. |

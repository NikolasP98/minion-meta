  <behavior>
    - GatewayClient.connect() opens the WebSocket, handles the connect.challenge event by invoking onChallenge(nonce), sends the connect request with the callback's result, and resolves with the server's HelloOk payload.
    - GatewayClient.request<T>(method, params) returns a Promise; rejects with timeout after requestTimeoutMs (default 15000ms).
    - On close, all pending requests reject with an error containing the close code.
    - When autoReconnect: true, a close that wasn't initiated by .close() schedules a reconnect with exponential backoff 800ms -> 15000ms (x1.7 per attempt).
    - When autoReconnect: false (default), close does NOT reconnect.
    - The same class works with globalThis.WebSocket (browser) AND with ws npm WebSocket (Node) — event wiring normalizes .on() vs addEventListener().
    - PROTOCOL_VERSION = 3 is exported from @minion-stack/shared root.
    - createNodeGatewayClient({ url, headers?, maxPayload?, onChallenge, ... }) returns a GatewayClient instance that uses ws under the hood.
    - Browser consumers importing only from '.' (root) do NOT pull ws into their bundle.
  </behavior>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local workstation -> npm registry | Publishing a signed, 2FA-authenticated package crosses from trusted dev machine to public registry. |
| @minion-stack/shared -> consumer runtimes | The published client code will run in browsers AND Node; a malicious dep chain here would affect hub + site users + paperclip agents. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02 | Tampering | packages/shared/src/gateway/client.ts — message deserialization | mitigate | Wrap JSON.parse(raw) in try/catch; silent-return on parse error (preserves existing protocol.ts behavior). Never cast frame without checking frame.type. Tested via unit test that injects malformed JSON. |
| T-07-03 | Denial of Service | GatewayClient pending request map | mitigate | Every request() sets a setTimeout that deletes the pending entry on timeout; close() calls flushPending(this.pending, err) to reject all outstanding. Tested via it('close() flushes pending...'). |
| T-07-04 | Denial of Service | Reconnect loop | mitigate | Backoff capped at 15000ms via Math.min(backoff * 1.7, 15000). autoReconnect defaults to false — Node consumers (paperclip) that should fail fast inherit the safe default. Tested via it('schedules reconnect with exponential backoff'). |
| T-07-05 | Spoofing | Stale socket delivers response to wrong generation | mitigate | generation counter increments per connect; all event handlers capture gen and early-return if this.generation !== gen. Pattern inherited from hub/paperclip. |
| T-07-06 | Information disclosure | Supply-chain: npm publish credentials | mitigate | Publish step is a human checkpoint requiring 2FA OTP. No automated publish. npm whoami verified before publish. |
| T-07-07 | Information disclosure | ws package CVE | accept | Pin ws ^8.0.0 as peerDep; consumers get version from their own lockfile. Monitor via Phase 8 npm audit CI. |
</threat_model>
